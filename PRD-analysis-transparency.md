# PRD - 分析过程透明化

## 1. Summary

为 TradingAgents-CN 添加**分析过程透明化**功能，让用户能看到完整的分析逻辑链（数据→分析→结论），从而判断分析结果是否合理、有效、可信。

## 3. Background

### 现状
- TradingAgents-CN 提供多智能体股票分析
- 分析结果只有最终结论（买入/卖出 + 置信度）
- **用户看不到分析过程**，无法判断结论是否可信

### 问题
```
用户心理：
"你告诉我买入，但我不知道你是怎么分析的"
"为什么你觉得这支股票值得买入？"
"我凭什么相信一个 AI 的判断？"
```

### 为什么现在解决
- 基础功能已完成（P0-P2 测试通过）
- 进入信任建立阶段
- 用户明确提出需求

## 4. Objective

### 目标
**让分析过程可见、可信、可验证**

### 关键结果 (SMART)
- KR1: 用户能在报告中看到**至少 3 个**关键分析依据（技术/基本面/情绪）
- KR2: 每个分析结论都有对应的**数据支撑说明**
- KR3: 用户可以**追溯**从原始数据到最终结论的逻辑链

### 价值
- **信任**: 用户基于理解而非盲从做决策
- **验证**: 用户可以质疑或验证分析逻辑
- **差异化**: 透明度成为产品竞争力

## 5. Market Segment(s)

### 目标用户
- **主要**: 有一定投资经验的个人投资者，希望理解 AI 分析逻辑
- **次要**: 量化研究者，验证 AI 分析方法论

### 用户约束
- 不需要金融专业背景也能理解
- 分析解释要简洁明了
- 不需要完整的 LLM 可解释性（那是不可能的）

## 6. Value Proposition(s)

### 用户收益
1. **理解** - 用户知道分析是怎么做出的
2. **验证** - 用户可以对照原始数据验证结论
3. **信任** - 基于理解的信任，而非盲目相信
4. **控制** - 用户感觉自己掌控了决策过程

### 解决的问题
- ❌ 以前: 黑盒分析，用户只能选择相信或不信
- ✅ 现在: 透明分析，用户可以验证和质疑

## 7. Solution

### 7.1 UX/UI 设计方向

#### 分析报告结构（新版）
```
📊 贵州茅台 (600519) 分析报告
├── 📋 分析概览
│   ├── 股票: 600519 贵州茅台
│   ├── 分析时间: 2026-04-10
│   └── 结论: 买入 (置信度 75%)
│
├── 🔍 分析过程（新增）
│   ├── 📈 技术分析
│   │   ├── 指标: MA5/MA10/MA20 多头排列 ✅
│   │   ├── RSI: 58.3 (中性偏强) ✅
│   │   ├── MACD: 金叉形成 ✅
│   │   └── 结论: 短期技术面看涨
│   │
│   ├── 📊 基本面分析
│   │   ├── PE: 28.5 (行业平均 32.1) ✅
│   │   ├── ROE: 23.4% (优秀) ✅
│   │   ├── 营收增长: +15.2% ✅
│   │   └── 结论: 基本面稳健
│   │
│   ├── 📰 情绪分析
│   │   ├── 利好: AI 硬件订单延伸至2028年
│   │   ├── 利好: TSMC 业绩验证供应链韧性
│   │   └── 结论: 行业情绪积极
│   │
│   └── 🎯 综合结论
│       ├── 技术面: 3/3 指标看涨
│       ├── 基本面: 3/3 指标合格
│       ├── 情绪面: 2/2 利好
│       └── 最终建议: 买入 (置信度高)
│
└── 💡 决策依据（新增）
    ├── 为什么买入？
    │   1. 技术形态确认机构吸筹
    │   2. 基本面优于行业平均
    │   3. 行业情绪积极
    │
    └── 风险提示
        └── 建议止损位: -6%
```

#### 页面布局建议
- **新增"分析过程"Tab** - 与"概览"、"详细报告"并列
- **每个分析模块折叠/展开** - 用户可深入查看细节
- **数据来源标注** - 明确告诉用户数据从哪来

### 7.2 关键功能

#### F1: 分析依据展示
- 技术指标结论及数据来源
- 基本面指标结论及数据来源
- 情绪分析结论及事件列表

#### F2: 逻辑链追溯
- 从原始数据 → 中间结论 → 最终建议
- 每步标注依据和置信度

#### F3: 结论解释
- "为什么建议买入？" 一键展开
- 自然语言解释决策逻辑

#### F4: 风险提示
- 明确标注风险等级
- 止损位置建议

### 7.3 技术实现

#### 后端
- 在 `reports` 数据中新增 `analysis_details` 字段
- 结构化存储每个分析师的结论和依据
- 保留原有的 markdown 报告作为补充

#### 前端
- 新增"分析过程"展示组件
- 折叠面板展示详情
- 数据可视化（指标仪表盘）

### 7.4 假设

1. ✅ 后端已经生成了各分析师的结论（只是没有保存/返回）
2. ✅ 前端可以解析和展示结构化数据
3. ⚠️ 需要确认 reports 数据结构是否支持扩展
